# 前后端联调完整指南

## 📋 前置准备

### 1. 本地启动 MySQL 和 Redis（Docker 一键启动）
```bash
# 启动 MySQL
docker run -d --name tiktok-mysql -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -e MYSQL_DATABASE=tiktok_auction \
  mysql:8.0

# 启动 Redis
docker run -d --name tiktok-redis -p 6379:6379 redis:7.2
```

### 2. 配置后端环境变量
进入 backend 目录：
```bash
cd backend
cp .env.example .env
```

编辑 .env 文件：
```env
PORT=3000
NODE_ENV=development
DATABASE_URL="mysql://root:123456@localhost:3306/tiktok_auction"
REDIS_URL="redis://localhost:6379/0"
JWT_SECRET=your_super_secret_jwt_key_here_change_in_production
JWT_EXPIRES_IN=7d
```

### 3. 安装后端依赖并初始化数据库
```bash
npm install
npm run prisma:generate
npm run prisma:migrate:dev --name init
npx ts-node src/scripts/init-data.ts
```

### 4. 启动后端服务
```bash
npm run start:dev
```
后端服务将在 http://localhost:3000 启动

## 🚀 完整联调流程步骤

### 步骤 1：启动商家端
```bash
cd frontend/merchant-admin
npm install
npm run dev
```
访问 http://localhost:5173，使用 test_merchant / 123456 登录

### 步骤 2：创建测试商品
1. 进入「商品管理」页面
2. 点击「创建商品」
3. 填写商品信息：
   - 商品名称：测试竞拍商品
   - 起拍价：0
   - 最小加价：1
   - 竞拍时长：60（秒，方便测试）
   - 自动延时：10
   - 延时阈值：5
4. 保存商品

### 步骤 3：初始化并开始竞拍
1. 在商品列表找到刚创建的商品
2. 点击「初始化竞拍」
3. 点击「开始竞拍」
4. 进入竞拍监控页面，观察倒计时

### 步骤 4：启动用户端
新开终端：
```bash
cd frontend/user-h5
npm install
npm run dev
```
访问 http://localhost:5174，使用 test_user / 123456 登录

### 步骤 5：用户端加入竞拍
1. 进入首页，看到正在竞拍的商品
2. 点击商品进入竞拍详情页
3. 点击「立即出价」按钮
4. 观察实时价格更新和倒计时

## 📡 WebSocket 实时消息说明

所有消息都通过 Socket.io 实时推送：

| 事件名 | 说明 |
|--------|------|
| price_update | 价格更新 |
| new_bid | 新出价 |
| countdown | 倒计时每秒推送 |
| auction_started | 竞拍开始 |
| auction_extended | 竞拍自动延时 |
| auction_ended | 竞拍结束 |
| auction_cancelled | 竞拍取消 |
