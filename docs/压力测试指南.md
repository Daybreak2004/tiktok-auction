# 直播竞拍系统 - 高并发压力测试指南

## 🎯 测试目标
验证系统在 100+ 用户同时出价场景下的稳定性，确保：
- 没有超卖
- 价格同步一致
- 无竞态条件
- 系统响应时间 < 100ms

---

## 🛠️ 推荐工具选择

### 方案一：Artillery（推荐，Node.js生态，简单易用）
- 专门针对 Node.js 后端优化
- 支持 WebSocket 压力测试
- 配置文件简单，YAML 格式
- 报告可视化

### 方案二：k6（现代高性能）
- Go 语言编写，性能极强
- JavaScript 写测试脚本
- 支持 1000+ 并发用户

## 🚀 Artillery 压力测试完整方案

### 1. 安装 Artillery
```bash
npm install -g artillery
artillery --version
```

### 2. 创建测试配置文件 `stress-test.yml`
```yaml
config:
  target: "http://localhost:3000"
  phases:
    - duration: 30
      arrivalRate: 10
      name: "预热阶段 - 每秒10个用户"
    - duration: 60
      arrivalRate: 50
      name: "高压阶段 - 每秒50个用户"
    - duration: 30
      arrivalRate: 10
      name: "冷却阶段"
  plugins:
    metrics-by-endpoint: {}
    statsd:
      host: "localhost"
      port: 8125

scenarios:
  - name: "高并发出价场景"
    flow:
      - post:
          url: "/user/auth/login"
          json:
            username: "test_user"
            password: "123456"
          capture:
            - json: "$.token"
              as: "userToken"
      - loop:
          - post:
              url: "/user/products/1/bid"
              headers:
                Authorization: "Bearer {{ userToken }}"
              json:
                price: "{{ $loopCount + 1 }}"
          - think: 1
          count: 20
```

### 3. 运行压力测试
```bash
artillery run stress-test.yml --output report.json
artillery report report.json
```

## 📊 WebSocket 实时连接压力测试

创建 `websocket-stress-test.js`：
```javascript
const io = require('socket.io-client');

const NUM_CLIENTS = 100;
const SERVER_URL = 'http://localhost:3000';
const TOKEN = 'your_jwt_token_here';

let connectedCount = 0;
let messageCount = 0;

console.log(`正在连接 ${NUM_CLIENTS} 个 WebSocket 客户端...`);

for (let i = 0; i < NUM_CLIENTS; i++) {
  const socket = io(SERVER_URL, {
    transports: ['websocket'],
    auth: { token: TOKEN }
  });

  socket.on('connect', () => {
    connectedCount++;
    console.log(`客户端 ${i} 已连接，总连接数: ${connectedCount}`);
    
    socket.emit('join_room', 'product:1');
  });

  socket.on('price_update', (data) => {
    messageCount++;
    console.log(`收到消息 #${messageCount}:`, data.currentPrice);
  });

  socket.on('disconnect', () => {
    console.log(`客户端 ${i} 断开连接`);
  });
}
```

运行 WebSocket 压力测试：
```bash
node websocket-stress-test.js
```

## ✅ 压力测试验证清单

测试完成后检查以下指标：

- [ ] 所有出价都成功，没有报错
- [ ] 最终价格 = 起拍价 + 出价次数 × 加价幅度
- [ ] 没有重复出价，没有跳价
- [ ] Redis 中的最终价格和数据库一致
- [ ] 所有客户端收到的价格完全同步
- [ ] 系统 CPU 使用率 < 80%
- [ ] 内存没有明显泄漏

## 🔍 常见问题排查

| 问题现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 价格跳变不一致 | 没有原子操作 | 确保 Lua 脚本正确执行 |
| 超卖/重复出价 | 分布式锁失效 | 检查锁的 TTL 和释放逻辑 |
| WebSocket 断开重连 | 服务器压力太大 | 增加服务器资源，优化心跳 |
