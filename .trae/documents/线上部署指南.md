# 直播竞拍系统 - 线上部署指南

## 一、云服务选型说明

本部署指南基于**火山引擎**云服务构建，也可平滑迁移至阿里云/腾讯云。

> ⚠️ 云部署不是必选项，按需选择，项目结束后记得关闭服务避免持续扣费。预估项目周期总花费约150元。

---

## 二、技术栈调整（适配云部署）

### 调整后的完整技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端-商家端 | React 18 + TypeScript 5 + Vite 5 + Ant Design 5 | PC管理后台 |
| 前端-用户端 | React 18 + TypeScript 5 + Vite 5 + Ant Design Mobile 5 | 移动端H5 |
| 后端 | NestJS 10 + Node.js 20 | 企业级Node.js框架 |
| 实时通信 | Socket.io 4 | WebSocket长连接 |
| ORM | Prisma 5 | 类型安全数据库访问 |
| 关系型数据库 | 火山引擎云数据库 MySQL 8.0 | 托管式MySQL服务 |
| 缓存/分布式锁 | 火山引擎云数据库 Redis 7.0 | 托管式Redis服务 |
| 云服务器 | 火山引擎 ECS | Ubuntu 22.04 LTS |
| 反向代理 | Nginx | 统一入口、SSL、静态资源托管 |
| 进程管理 | PM2 | Node.js服务守护进程 |

## 三、第一步：火山引擎账号准备

### 1. 注册与实名认证
1. 访问火山引擎控制台：https://console.volcengine.com/auth/login
2. 完成账号注册和实名认证
3. 进入费用中心充值，建议预存50-100元（按量付费很便宜）

---

## 四、第二步：创建云服务器 ECS

### 配置选择
| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 计费方式 | 按量付费 | 项目结束后释放，成本可控 |
| 地域 | 华北2（北京） | 选择离你近的区域 |
| 实例规格 | 2C4G 起步，推荐4C8G | 最低配置就能跑 |
| 操作系统 | Ubuntu 22.04 LTS | 文档最完善 |
| 系统盘 | 40GB SSD | 足够用 |
| 登录方式 | SSH密钥对（推荐）或密码 | 密钥更安全 |

### 创建步骤
1. 进入「产品与服务」→「云服务器 ECS」
2. 点击「创建实例」
3. 按上述配置选择，完成创建
4. 在「实例与镜像」列表中看到你的ECS公网IP

### 连接服务器
```bash
# 使用SSH连接
ssh root@<你的ECS公网IP>
```

---

## 五、第三步：创建云数据库 MySQL

### 前置准备：私有网络 VPC
1. 进入「产品与服务」→「私有网络」
2. 创建默认VPC：`172.31.0.0/16`
3. 创建至少1个子网，选择和ECS相同的可用区

### 创建MySQL
1. 进入「产品与服务」→「关系型数据库」→「云数据库MySQL版」
2. 点击「创建实例」
3. 基础配置：
   - 数据库版本：MySQL 8.0
   - 实例规格：1核1GB（入门版足够）
   - 存储：20GB
4. 网络配置：选择刚才创建的VPC和子网
5. 设置白名单：把你的ECS内网IP加进去（这样ECS才能访问MySQL）
6. 设置root密码，完成创建

### 获取连接信息
创建完成后，你会得到：
- 内网连接地址（ECS内网访问，速度快不花钱）
- 端口：默认3306
- 用户名：root
- 密码：你设置的密码

---

## 六、第四步：创建云数据库 Redis

### 创建Redis
1. 进入「产品与服务」→「NoSQL数据库」→「缓存数据库Redis版」
2. 点击「创建实例」
3. 配置选择：
   - 版本：Redis 7.0
   - 规格：1GB（最小规格约0.66元/小时）
   - 网络：和MySQL、ECS同一个VPC子网
4. 设置白名单，加入ECS内网IP
5. 设置密码，完成创建

### 获取连接信息
- 内网连接地址
- 端口：6379
- 密码：你设置的密码

## 七、第五步：ECS环境初始化

连接到你的ECS服务器后，执行以下命令：

### 1. 更新系统
```bash
apt update && apt upgrade -y
```

### 2. 安装 Node.js 20
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
node -v  # 验证版本
npm -v
```

### 3. 安装 PM2（进程管理）
```bash
npm install -g pm2
pm2 --version
```

### 4. 安装 Nginx
```bash
apt install -y nginx
nginx -v
systemctl enable nginx
systemctl start nginx
```

### 5. 安装 Git
```bash
apt install -y git
git --version
```

### 6. 安装 Docker（可选，更方便）
```bash
apt install -y docker.io
systemctl enable docker
systemctl start docker
```

## 八、第六步：项目部署

### 项目目录结构规划
```
/opt/tiktok/
├── merchant-admin/    # 商家端前端构建产物
├── user-h5/          # 用户端H5构建产物
└── backend/          # 后端服务
```

### 1. 上传代码到服务器
你可以用Git克隆，或者用SCP上传：
```bash
# 方式1：Git克隆（推荐）
cd /opt
git clone <你的代码仓库地址> tiktok

# 方式2：SCP从本地上传
scp -r ./tiktok root@<ECS_IP>:/opt/
```

### 2. 构建前端项目

**商家端构建：**
```bash
cd /opt/tiktok/frontend/merchant-admin
npm install
npm run build
# 产物输出到 dist/ 目录
```

**用户端H5构建：**
```bash
cd /opt/tiktok/frontend/user-h5
npm install
npm run build
# 产物输出到 dist/ 目录
```

### 3. 后端部署
```bash
cd /opt/tiktok/backend
npm install

# 配置环境变量
cp .env.example .env
nano .env
```

编辑 `.env` 文件，填入你的云数据库信息：
```env
# 服务端口
PORT=3000

# MySQL连接字符串（用内网地址）
DATABASE_URL="mysql://root:你的密码@你的MySQL内网地址:3306/tiktok_auction"

# Redis连接字符串（用内网地址）
REDIS_URL="redis://:你的Redis密码@你的Redis内网地址:6379/0"

# JWT密钥
JWT_SECRET=这里填一个随机长字符串
```

执行数据库迁移：
```bash
npx prisma migrate deploy
```

用PM2启动后端服务：
```bash
pm2 start dist/main.js --name tiktok-backend
pm2 save
pm2 startup
```

## 九、第七步：Nginx 配置

创建Nginx配置文件 `/etc/nginx/sites-available/tiktok`：

```nginx
server {
    listen 80;
    server_name <你的ECS公网IP>;  # 后续换成域名

    # 商家端前端
    location /merchant/ {
        alias /opt/tiktok/frontend/merchant-admin/dist/;
        try_files $uri $uri/ /merchant/index.html;
    }

    # 用户端H5
    location /h5/ {
        alias /opt/tiktok/frontend/user-h5/dist/;
        try_files $uri $uri/ /h5/index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket支持
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # WebSocket超时设置长一点
        proxy_read_timeout 86400;
        proxy_send_timeout 86400;
    }
}
```

启用配置：
```bash
ln -s /etc/nginx/sites-available/tiktok /etc/nginx/sites-enabled/
nginx -t  # 测试配置
systemctl reload nginx
```

## 十一、视频直播部分（可选）

如果需要视频直播功能，推荐用开源方案快速搭建，不需要域名也能跑：

### 使用 SRS 开源流媒体服务器
```bash
# 在ECS上快速部署SRS
git clone https://github.com/ossrs/srs.git
cd srs/trunk
./configure --use-docker
docker run -d -p 1935:1935 -p 1985:1985 -p 8080:8080/udp \
  --name srs ossrs/srs:5
```

- 推流地址：`rtmp://<ECS_IP>/live/stream1`
- 播放地址：`http://<ECS_IP>:8080/live/stream1.m3u8`

用 OBS Studio 就可以开始推流了！

## 十二、成本控制与安全提醒

### 成本控制
- 所有云服务都选「按量付费」
- 项目演示完成后，及时释放ECS、MySQL、Redis实例
- 预估总花费：项目周期约150元以内

### 安全建议
- 不要在代码里硬编码密码，用环境变量
- 定期用 `pm2 monit` 监控服务状态
- 配置防火墙只开放必要端口：80(HTTP)、22(SSH)
