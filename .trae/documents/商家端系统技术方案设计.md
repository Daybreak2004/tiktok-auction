# 直播竞拍系统 - 商家端（PC管理后台）技术方案设计

## 一、商家端系统概述

### 1.1 系统定位
商家端是面向主播/商家的PC端管理后台，用于完成竞拍商品的全生命周期管理，包括商品上架、规则配置、竞拍监控、订单管理等核心功能。

### 1.2 核心用户角色
- **商家/主播**：创建和管理自己的直播间、发布竞拍商品、监控竞拍过程、处理订单

### 1.3 核心功能清单
| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 认证模块 | 商家登录、JWT认证、权限控制 | P0 |
| 商品管理 | 商品列表、商品详情、创建商品、编辑商品、取消竞拍 | P0 |
| 竞拍规则配置 | 起拍价、加价幅度、封顶价、竞拍时长、自动延时规则 | P0 |
| 直播间管理 | 创建直播间、开启/关闭直播、关联当前竞拍商品 | P1 |
| 实时监控 | 实时查看当前竞拍状态、出价动态、在线人数 | P1 |
| 订单管理 | 订单列表、订单详情、订单状态更新 | P0 |
| 数据统计 | 竞拍历史数据、成交统计 | P2 |

## 四、核心页面详细设计

### 4.1 登录页 (/login)
**功能点：**
- 用户名/密码输入
- 表单校验
- 登录状态持久化（LocalStorage存Token）
- 登录成功跳转到仪表盘

**UI组件：**
- Ant Design Form + Input + Button
- 品牌Logo展示
- 简洁的登录卡片布局

### 4.2 仪表盘 (/dashboard)
**功能点：**
- 统计卡片：今日竞拍数、总成交额、进行中竞拍数
- 进行中竞拍列表（实时刷新）
- 最近成交订单
- 快捷操作：快速创建商品

**实时特性：**
- 通过WebSocket接收当前竞拍的状态更新
- 价格变化实时高亮提示

### 4.3 商品列表页 (/products)
**表格列：**
| 列名 | 说明 |
|------|------|
| 商品封面 | 缩略图展示 |
| 商品名称 | 商品标题 |
| 竞拍状态 | 待开始/竞拍中/已结束/已取消（标签颜色区分） |
| 当前价格 | 实时价格 |
| 起拍价 | 配置的起拍价 |
| 创建时间 | 商品创建时间 |
| 操作 | 查看/编辑/开始竞拍/取消竞拍 |

**筛选条件：**
- 状态筛选下拉框
- 时间范围筛选
- 商品名称搜索

### 4.4 创建/编辑商品页 (/products/create /products/edit/:id)
**分步骤表单：**

**Step 1: 基础信息**
- 商品名称（必填）
- 商品描述（富文本/多行文本）
- 商品图片上传（支持多图，拖拽排序）
- 封面图选择

**Step 2: 竞拍规则配置**
- 起拍价：默认0元，支持自定义
- 最小加价幅度：默认1元，可配置
- 封顶价：可选，不填表示无封顶
- 竞拍时长：单位秒，默认300秒（5分钟）
- 自动延时时长：默认15秒
- 延时触发阈值：结束前N秒内出价触发延时，默认10秒

**Step 3: 确认提交**
- 预览商品信息和规则
- 提交保存

### 4.5 竞拍监控页 (/auction/:id)
**核心功能区域：**

**左侧面板 - 商品信息**
- 商品大图展示
- 商品名称、描述
- 竞拍规则概览

**中间面板 - 实时竞拍**
- 大字体显示当前价格（价格变化时动画高亮）
- 倒计时组件（精确到秒，红色警示）
- 实时出价动态流（最新出价消息滚动）
- 操作按钮：【立即开始竞拍】【手动结束竞拍】【取消竞拍】

**右侧面板 - 实时排行榜**
- Top N出价者列表
- 显示头像、昵称、出价金额、出价时间
- 第一名高亮显示

**实时通信：**
- 建立WebSocket连接，加入对应商品房间
- 接收消息类型：
  - `price_update` - 价格更新
  - `new_bid` - 新出价
  - `countdown` - 倒计时推送
  - `auction_ended` - 竞拍结束通知

### 4.6 订单列表页 (/orders)
**表格列：**
| 列名 | 说明 |
|------|------|
| 订单号 | 唯一订单标识 |
| 商品信息 | 商品名称+缩略图 |
| 成交用户 | 用户昵称+头像 |
| 成交价格 | 最终成交价 |
| 订单状态 | 待支付/已支付/已发货/已完成/已取消 |
| 成交时间 | 竞拍结束时间 |
| 操作 | 查看详情 |

**筛选条件：**
- 订单状态筛选
- 时间范围筛选

### 4.7 订单详情页 (/orders/:id)
**信息展示：**
- 订单基本信息（订单号、创建时间、状态）
- 商品信息卡片
- 成交用户信息
- 出价历史回溯（完整出价记录）
- 状态流转时间线
- 操作按钮：标记发货等

## 五、API接口设计

### 5.1 认证模块
```typescript
POST /api/merchant/auth/login
  Request: { username: string, password: string }
  Response: { token: string, userInfo: MerchantUser }

GET /api/merchant/auth/profile
  Headers: Authorization: Bearer <token>
  Response: MerchantUser
```

### 5.2 商品管理模块
```typescript
GET /api/merchant/products
  Query: { page, pageSize, status?, keyword? }
  Response: { list: Product[], total: number }

POST /api/merchant/products
  Request: CreateProductDto
  Response: Product

GET /api/merchant/products/:id
  Response: Product

PUT /api/merchant/products/:id
  Request: UpdateProductDto
  Response: Product

POST /api/merchant/products/:id/start
  Response: { success: boolean }

POST /api/merchant/products/:id/cancel
  Response: { success: boolean }
```

### 5.3 订单管理模块
```typescript
GET /api/merchant/orders
  Query: { page, pageSize, status? }
  Response: { list: Order[], total: number }

GET /api/merchant/orders/:id
  Response: OrderDetail

PUT /api/merchant/orders/:id/status
  Request: { status: string }
  Response: Order
```

## 六、状态管理设计 (Zustand)

### 6.1 useAuthStore
```typescript
interface AuthState {
  token: string | null
  user: MerchantUser | null
  isLoggedIn: boolean
  login: (username: string, password: string) => Promise<void>
  logout: () => void
}
```

### 6.2 useProductStore
```typescript
interface ProductState {
  productList: Product[]
  currentProduct: Product | null
  loading: boolean
  fetchProducts: () => Promise<void>
  fetchProductDetail: (id: string) => Promise<void>
}
```

## 七、关键交互细节

### 7.1 图片上传
- 使用Ant Design Upload组件
- 支持拖拽上传
- 图片预览、删除、排序
- 上传前压缩，限制单图5MB以内

### 7.2 实时竞拍监控体验
- 价格更新时数字跳动动画
- 新出价消息从底部滑入
- 倒计时最后10秒红色闪烁警示
- 声音提示（可选）

### 7.3 权限控制
- 路由守卫：未登录用户自动跳转/login
- 基于JWT的身份认证
- Token过期自动刷新/跳转登录页
